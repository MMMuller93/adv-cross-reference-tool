### N-CEN Verdict

N-CEN is **superior for the (registrant → adviser → sub-adviser → CRD) mapping** because:
1. CRD numbers are structured XML fields, not prose to be parsed
2. LEIs are provided for direct cross-reference to ADV database
3. Affiliation and country flags enable downstream filtering

N-CEN is **useless for PM identity** — it contains zero portfolio manager names, titles, or biographies. It is a fund census of fund-level metadata, not individual-level investment personnel.

**The answer is: use N-CEN to build the (fund → adviser → CRD) graph, and use N-1A/N-2 to extract PM names and bios.** These two sources are complementary, not competitive.

## 6. Recommendation

### Source Strategy for (Fund → PM Identity) Lookup

**Tier 1 — N-CEN XML (structured, annual):**
- Build fund → adviser → sub-adviser mapping with CRDs
- Use `<investmentAdviserCrdNo>` and `<subAdviserCrdNo>` to join to the ADV `advisers_enriched` table
- Machine-parseable with a simple XML walker; no LLM needed
- Limitations: No PM names. Only tells you *which entity* manages the fund, not which person.

**Tier 2 — N-1A/N-2 "Portfolio Management" section (semi-structured, annual):**
- Source of actual PM names, roles, titles, tenure, and biographical career summaries
- For ~80% of N-1A filers: regex-parseable. Pattern for large fund families:
  - Name + "(Role)" + "has managed the fund since YEAR" — extractable with ~50-line parser
  - T. Rowe: HTML `<table>` in Name/Title/Since/Joined columns
  - Baron: labeled "Portfolio Manager." heading then name sentence
  - Fidelity: repeated name block; filter to first unique occurrence
- For N-2 (closed-end/interval) filers like ARK and DXYZ: LLM extraction required; no consistent heading, PM identity buried in corporate structure prose
- **SAI exhibit** (often embedded in same 485BPOS file) adds compensation structure, other accounts managed, and shares owned — higher-resolution data than fund summary

**Recommended Pipeline:**
1. Query N-CEN XML per registrant CIK → extract `investmentAdviserCrdNo` + `subAdviserCrdNo` → JOIN to `advisers_enriched` (ADV DB)
2. Pull 485BPOS/N-2 most recent filing per registrant → extract "Portfolio Management" section → regex for structured filers, LLM for N-2/boutique filers
3. For subadvised funds, also pull the sub-adviser's own Form ADV Part 2A (available via IAPD/EDGAR) → PM bios at the sub-adviser level are in ADV Part 2A Item 9/Appendix 1, which is *more* detailed than the fund's prospectus

**Contact path**: N-1A does not contain PM emails or direct contact info. The path is: PM name → Form ADV Part 2B (brochure supplement, filed by investment advisers) → includes professional background, education, certifications. This is the richest individual-level disclosure in the EDGAR ecosystem for registered advisers.
